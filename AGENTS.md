## What this project is

Self-hosted portal for a home media setup: an application dashboard kept in sync by a Kubernetes
operator watching Gateway API `HTTPRoute`s, Matrix notifications for *arr/Jellyseerr webhooks, a
Matrix bot, and downloads/library statistics synced from the *arr apps, Jellyfin and Jellystat.
See [README.md](README.md) for the full feature list and configuration reference.

## Stack

- Backend: Quarkus 3.x + Kotlin (Java 21), Hibernate Panache **Reactive**, PostgreSQL, Flyway
- Frontend: **Vue 3** + TypeScript, Vite, Pinia, Vue Router, TanStack Vue Query, Tailwind CSS 4,
  Chart.js — served by the backend via Quarkus Quinoa
- Matrix: Trixnity client library
- Operator: Java Operator SDK + Fabric8 (Gateway API models)
- Build: Maven multi-module (use `./mvnw`), npm for the frontend

## Modules

| Module | Purpose |
|---|---|
| `app/` | Main Quarkus app: REST API, webhooks, Matrix bot, stats sync + the Vue UI in `app/src/main/webui/` |
| `operator/` | Kubernetes operator syncing annotated `HTTPRoute`s to the app's registry — contract in [operator/README.md](operator/README.md) |
| `synapse-devservice/` | Custom Quarkus dev service spinning up a Synapse (Matrix) homeserver for dev/tests |
| `charts/homelab-manager/` | Helm chart deploying app + operator |

Backend packages follow `org.hoohoot.homelab.manager.<feature>.<layer>` with layers `api` (REST
resources), `domain` (business logic), `infra` (persistence, clients). Features: `applications`,
`notifications`, `statistics`, `library`, `cleanup`, `finances`, `members`, `problems`, plus
`shared` (REST clients, Matrix/Trixnity, security, time utilities).

## Commands

```bash
./mvnw verify                    # full build + all tests (this is what CI runs)
./mvnw -pl app quarkus:dev       # app + web UI with dev services (Keycloak, Postgres, WireMock, Synapse)
./mvnw -pl operator quarkus:dev  # operator, against your local kubeconfig
./mvnw -pl app test              # backend tests only

# Frontend (from app/src/main/webui/, Node 20 — see .nvmrc)
npm run dev          # Vite dev server (regenerates the Orval API client first)
npm run test         # Vitest
npm run test:ci      # ESLint + Vitest (what mvn verify runs via Quinoa)
npm run lint:fix     # ESLint autofix
npm run format       # Prettier
npm run type-check   # vue-tsc strict
```

CI (`.github/workflows/`): `mvn verify -B` on JDK 21 + JaCoCo coverage report on PRs, Helm
lint/template, Docker build. Releases are cut by release-please.

## Conventions

- **Conventional Commits** (`feat:`, `fix:`, `refactor:`, `ci:`, ...) — release-please derives
  versions and the changelog from them.
- Backend tests: JUnit 5 + `@QuarkusTest`, REST Assured (Kotlin extensions), AssertJ, WireMock
  for external APIs. Tests live in `app/src/test/kotlin/...` mirroring the main packages.
- Frontend tests: Vitest + @vue/test-utils, in `app/src/main/webui/src/test/`.
- Frontend formatting is Prettier's job (single quotes, no semicolons); ESLint only checks
  correctness. `src/api/` is **generated by Orval** from the backend OpenAPI schema — never edit
  it by hand, it is regenerated on build/dev.
- DB migrations: Flyway SQL files in `app/src/main/resources/db/migration/`, named
  `V<N>__<description>.sql`, applied at startup. Never modify an existing migration; add a new one.

## Gotchas

- The DB is accessed through **two** drivers: JDBC (`DB_JDBC_URL`, Flyway only) and reactive
  (`DB_REACTIVE_URL`, the app). Both must point to the same database.
- App ⇄ operator auth is a shared key (`OPERATOR_API_KEY`) sent as `X-Api-Key` on
  `/api/operator/*`; if unset on the app side, operator calls are rejected 401 (fail-closed).
  No OIDC involved on that path.
- Dev mode auto-provisions everything via dev services: Keycloak (users `alice`/`bob`),
  PostgreSQL, WireMock (stubs for Sonarr/Radarr/Lidarr/Bazarr/Jellyfin/Giphy... defined in
  `app/src/main/resources/wiremock/`), and Synapse via the custom `synapse-devservice` module.
- The operator sets no finalizers on `HTTPRoute`s (they belong to Flux/Helm); deletions are
  caught by the periodic sweep. It only deletes portal apps marked `managedBy=operator`.
- All runtime configuration is environment variables (Quarkus `some.key` → `SOME_KEY`); the full
  table is in [README.md](README.md).
